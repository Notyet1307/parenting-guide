# Project Spec: 孕期导航仪 (Parenting Navigation)

## 1. 核心任务 (JTBD - Jobs To Be Done)

我们究竟在解决什么问题？

> **当** 准爸妈面对复杂的孕期检查和准备工作感到手足无措时，
> **他们想要** 一个清晰的、按周次划分的"行动指令清单"，
> **以便于** 按部就班地完成准备，消除未知带来的焦虑，并减少因为"谁该干什么"而产生的争吵。

*   **Done State (爽点)**：打开网页，一眼看到"这周要做什么"，点击"完成"，感到即使是第一次当父母，一切也在掌控之中。

## 2. 核心功能 (P0 Features - MVP Scope)

为了验证核心假设，第一版（MVP）**必须**包含且**仅**包含以下功能：

### P0-1: 智能时间轴 (The Timeline)
*   **输入**：用户输入预产期 (Due Date)。
*   **输出**：自动生成从"当前孕周"到"出生"的完整时间轴。
*   **展示**：
    *   当前是第几周？宝宝现在多大了（水果比喻）？
    *   本周重点：一句话总结（如"该做大排畸了"）。

### P0-2: 角色化任务清单 (Role-based Tasks)
*   **核心逻辑**：任务自动分配给"准爸"和"准妈"。
*   **准爸任务**：偏执行（如"购买叶酸"、"预约建档"、"准备待产包-证件类"）。
*   **准妈任务**：偏体验与监测（如"记录体重"、"数胎动"、"感受第一次胎动"）。
*   **交互**：简单的打钩完成（Check-off），双方可见进度。

### P0-3: "标准答案"知识卡片 (The "Truth" Cards)
*   **针对痛点**：减少百度/小红书带来的杂音和焦虑。
*   **功能**：针对本周任务，提供短小精悍的"标准操作指南"（如"唐筛和无创DNA怎么选？直接选无创"）。

## 3. 不做清单 (The Not-To-Do List)

为了保证 3 周内上线，我们**坚决不做**：
*   ❌ **不做社区/论坛**：不需要用户讨论，我们提供标准答案。
*   ❌ **不做商城/交易**：只提供购物清单，不直接卖货。
*   ❌ **不做复杂的健康数据分析**：不画复杂的体重/血糖曲线，只做简单记录。
*   ❌ **不做原生 App**：只做响应式 Web (Mobile Web)。

## 4. 风险预判 (Pre-mortem)

*   **风险 1**：用户觉得"内容太通用，不适合我的具体情况"。
    *   *对策*：MVP 虽通用，但语气要极其笃定（Expert Tone），建立信任感。
*   **风险 2**：准爸爸不配合，只有妈妈在用。
    *   *对策*：设计"分享给爸爸"的强制/诱导机制，或者通过微信通知提醒爸爸"这周该你干活了"。

## 5. 数据模型概览 (Data Model - Draft)
*   `User`: id, role (mom/dad), due_date
*   `Timeline`: week_num, baby_info, mom_symptoms
*   `Task`: id, week_num, content, assignee (mom/dad), is_completed
